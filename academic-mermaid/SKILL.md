# 学术规范 Mermaid 流程图生成指南

当用户要求根据文本或代码上下文生成 Mermaid 流程图，且提及“学术”、“学位论文”、“算法流程图”、“示意图”等要求时，请严格遵循以下工作流：

## 1. 核心流程提取与凝练 (Schematic Extraction)
- **抓大放小（重示意，轻细节）**：提取核心的数据流或处理节点（例如：采集 → 预处理 → 分割 → 后处理）。摒弃代码底层的实现细节（如具体的线程锁机制、多余的临时变量等），除非用户明确要求生成详细的软件架构图。
- **扁平化结构**：尽量避免使用复杂的、嵌套的 `subgraph`。追求最短判定路径，减少线条交叉，使视觉呈现直截了当。

## 2. 学术样式规范 (Academic Styling)
- **纯黑白无背景**：禁止使用任何彩色或灰色的 `fill` 背景，以符合传统学术论文的印刷标准。需在图表顶部统一声明默认样式（覆盖掉 Mermaid 的默认主题配置），明确设定基础主题，并将所有连线文本的背景设置为纯白 `#ffffff` 以防部分渲染器深色模式报错：
  ```mermaid
  %%{init: {'theme': 'base', 'themeVariables': { 'edgeLabelBackground':'#ffffff'}}}%%
  graph TD
      classDef default fill:#ffffff,stroke:#000,stroke-width:1.5px,color:#000;
  ```
- **沿用标准算法框图形状**：
  - **开始/结束/外部实体**：使用圆角矩形或胶囊形，语法为 `(["实体名称"])`
  - **处理/普通节点**：使用直角矩形，语法为 `["处理操作"]`
  - **判断/分支条件**：使用菱形，语法为 `{"判定条件"}`
  - **数据读写/IO**：使用平行四边形，语法为 `[/"数据内容"/]`

## 3. 防错语法与排版优化 (Syntax & Layout Optimization)
- **强制使用双引号**：Mermaid 解析器对中英文混排标点和 HTML 标签（如 `<br/>`、`()`）极其敏感，容易报 `Parse Error`。
- **节点文本防护**：所有节点内部的展示文本必须用双引号包裹。**正确示例**：`NodeA["步骤A<br/><small>补充</small>"]`。
- **连线标签防护**：所有连线上的描述文字必须用双引号包裹。**正确示例**：`-->|"发送数据"|`。
- **双向箭头重叠避免**：当两个节点相互引用（导致连线标签重叠在正中间）时，强制采用增加线长（例如 `A --->|"长线"| B`）配合折行的策略错开交叠位置。例如要求长文字通过 `<br/>` 进行切分，不要水平占据过多面积。
- **同源多出线交叉避免**：当一个节点同时向两个目标连线时，Mermaid Dagre 引擎按声明顺序从左到右分配出线端口。若两条线的目标节点左右位置与声明顺序相反，线条就会交叉。**解决方法**：调整连线的声明顺序，使其与目标节点在布局中的左右位置一致。
- **密集区域拆数据节点**：当同一节点有多条带长标签的出线且标签互相遮挡时，将连线标签改为独立的中转数据节点（平行四边形 `[/"文本"/]`），原连线改为无标签箭头，从根本上消除标签重叠。

## 4. 验证与输出 (Validation & Output)
- 检查产出的代码：所有的 `fill` 是否都是 `#ffffff`？是否有漏加引号的节点导致渲染风险？
- 将最终梳理好的 Mermaid 代码块保存至用户要求的文件（如 `docs/*.md`），或在代码块中输出。
